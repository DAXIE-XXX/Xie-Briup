# 车牌识别PyQt应用程序使用说明

## 功能特性

1. **摄像头识别**：支持调用摄像头实时识别车牌
2. **图片上传识别**：支持上传图片进行车牌识别
3. **车牌数据库比对**：识别后自动与数据库中的注册车牌进行比对
4. **车牌注册管理**：可以注册、查看、删除车牌信息
5. **比对结果显示**：比对通过显示"车牌号请通过"，未注册显示警告
6. **模型选择管理**：支持从本地或模型服务器选择不同的识别模型
7. **现代化UI界面**：优化的界面设计，更好的用户体验

## 安装依赖

```bash
pip install -r requirements_qt.txt
```

或者手动安装：

```bash
pip install PyQt5 opencv-python numpy onnxruntime Pillow sqlalchemy
```

## 运行程序

```bash
python car_plate_recognition_app.py
```

## 使用说明

### 1. 选择模型
- 在顶部"模型管理"区域的下拉框中选择要使用的识别模型
- 模型列表会自动从本地 `car_num_models` 目录和模型服务器（如果可用）加载
- 显示格式：`模型名称 (来源)`，来源可以是"本地"或"服务器"
- 点击"刷新列表"按钮可以重新加载模型列表
- 当前使用的模型会显示在状态标签中

### 2. 启动摄像头识别
- 点击"📷 启动摄像头"按钮
- 摄像头画面会实时显示在左侧图像区域
- 点击"🔍 识别车牌"按钮进行识别

### 3. 上传图片识别
- 点击"📁 上传图片"按钮
- 选择包含车牌的图片文件
- 图片会显示在左侧图像区域
- 点击"🔍 识别车牌"按钮进行识别

### 4. 车牌注册
- 在右侧"车牌注册"区域输入车牌号
- 可选输入车主姓名和备注
- 点击"✅ 注册车牌"按钮完成注册
- 注册的车牌会显示在下方的列表中

### 5. 车牌比对
- 识别车牌后，系统会自动与数据库中的注册车牌进行比对
- 如果车牌已注册，会显示"车牌号请通过"的绿色提示
- 如果车牌未注册，会显示"车牌未注册"的橙色警告

### 6. 删除车牌
- 在已注册车牌列表中选中要删除的车牌
- 点击"❌ 删除选中"按钮
- 确认删除操作

## 模型文件要求

### 本地模型
程序会自动查找 `car_num_models` 目录下的所有模型配置。每个模型目录应包含：
- `config.json` - 模型配置文件
- `yolo_best.onnx` - YOLO检测模型（或配置中指定的文件名）
- `ocr_best.onnx` - OCR识别模型（或配置中指定的文件名）
- `ppocr_keys_v1.txt` - OCR字典文件（或配置中指定的文件名）

如果找不到任何模型，程序会尝试使用根目录下的默认模型文件：
- `yolo_best.onnx` - YOLO检测模型
- `ocr_best.onnx` - OCR识别模型
- `ppocr_keys_v1.txt` - OCR字典文件

### 模型服务器
程序支持从模型服务器获取模型列表。默认服务器地址为 `http://localhost:8000`。

要使用模型服务器功能：
1. 确保Flask服务器正在运行（运行 `python flask_server.py`）
2. 程序启动时会自动尝试连接服务器获取模型列表
3. 如果服务器不可用，程序会只使用本地模型

可以在代码中修改 `self.server_url` 来更改服务器地址。

## 数据库

程序使用SQLite数据库存储注册的车牌信息，数据库文件为 `registered_plates.db`。

数据库表结构：
- `id`: 主键
- `plate_number`: 车牌号（唯一）
- `owner_name`: 车主姓名
- `register_time`: 注册时间
- `notes`: 备注信息

## 注意事项

1. 确保摄像头权限已开启
2. 确保模型文件路径正确
3. 识别准确度取决于模型质量和图片质量
4. 车牌号比对是精确匹配，请确保识别结果准确

## 故障排除

1. **摄像头无法打开**：检查摄像头是否被其他程序占用
2. **模型加载失败**：检查模型文件路径和文件是否存在
3. **识别失败**：检查图片是否包含清晰的车牌
4. **数据库错误**：检查是否有写入权限
